## 1. 硬件接线(断电接线!)

| W25Q64 模块 | STM32 | 备注 |
|---|---|---|
| CS(片选) | PA4 | |
| CLK / SCK | PA5 | |
| DO(数据输出)| PA6 (MISO) | 模块的"输出"接 MCU 的"输入" |
| DI(数据输入)| PA7 (MOSI) | |
| VCC | **3.3V** | ⚠️ 接 5V 会烧芯片 |
| GND | GND | |

> 如果你的模块丝印是 MISO/MOSI 而不是 DO/DI:MISO→PA6,MOSI→PA7,同名直连。


`USER CODE BEGIN 2` 区(阶段①的打印之后)追加:

```c
  /* ---- 阶段② W25Q64 自检 ---- */
  printf("[App] W25Q64 JEDEC ID = 0x%06lX\r\n", (unsigned long)W25_ReadJEDEC());

  if (W25_Init() != 0) {
    printf("[App] W25Q64 NOT found! Check wiring.\r\n");
  } else {
    static uint8_t tx_buf[300], rx_buf[300];
    uint32_t addr = W25_TEST_ADDR + 200;   /* 故意不对齐页边界,验证跨页写 */
    uint16_t i;
    int      ok = 1;

    for (i = 0; i < sizeof(tx_buf); i++) {
      tx_buf[i] = (uint8_t)(i ^ 0xA5);
    }

    if (W25_EraseSector(W25_TEST_ADDR) != 0)            { ok = 0; }
    if (ok && W25_Write(addr, tx_buf, sizeof(tx_buf)))  { ok = 0; }
    if (ok && W25_Read(addr, rx_buf, sizeof(rx_buf)))   { ok = 0; }
    if (ok && memcmp(tx_buf, rx_buf, sizeof(tx_buf)))   { ok = 0; }

    printf("[App] W25Q64 erase/write/read test: %s\r\n", ok ? "PASS" : "FAIL");
  }
```

> 测试点设计:300 字节从"扇区内偏移 200"写起,必然跨越 256B 页边界——专门验证 `W25_Write` 的跨页拆分逻辑,这正是阶段③写固件时的真实场景。

## 4. 验收标准

串口输出:

```
[App] Firmware v1.0 running.
[App] W25Q64 JEDEC ID = 0xEF4017
[App] W25Q64 erase/write/read test: PASS
```

1. JEDEC ID 必须是 **0xEF4017**(Winbond W25Q64);
2. 测试打印 **PASS**;
3. 多按几次复位,每次都 PASS(排除接触不良)。

## 5. 常见坑

| 现象 | 原因 |
|---|---|
| ID = 0x000000 | MISO(PA6↔DO)没接通,或 CS 一直没拉低(PA4 接错) |
| ID = 0xFFFFFF | 模块没上电 / GND 没共地 / CLK、MOSI 接错 |
| ID 时对时错 | 杜邦线过长 9MHz 不稳 → CubeMX 里把 SPI1 Prescaler 从 8 改成 16(4.5MHz)重新生成 |
| ID 正确但读写 FAIL | 写前没擦除(Flash 只能 1→0,擦除后才全 0xFF),或写保护引脚 WP 被拉低 |
| 第一次 PASS 之后 FAIL | 没先擦除就重复写同一区域 |

## 6. 这一步学到的 Flash 特性(阶段③④会反复用)

- **擦除才能写**:NOR Flash 位只能 1→0,写前必须把所在 4KB 扇区整个擦成 0xFF;
- **按页编程**:一次最多写 256B 且不能跨页,所以 `W25_Write` 内部做了拆分;
- **擦写都要等 BUSY**:芯片内部操作期间读 SR1 的 BUSY 位轮询,这就是 `wait_busy()`;
- 阶段③接收固件时的节奏:每收满 4KB → 擦一个下载区扇区 → 写入,如此循环。
