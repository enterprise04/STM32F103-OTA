# STM32F103 串口 IAP/OTA 升级系统 —— 总体设计

> 硬件:STM32F103C8T6 最小系统板(64KB Flash / 20KB RAM)+ W25Q64 + USB 转串口 + OLED
> 环境:Keil MDK + STM32CubeMX
> 日期:2026-06-12

## 1. 总体架构

```
PC 上位机(Python + pyserial,模拟云端服务器)
        │  自定义帧协议 + CRC32,经 USB 转串口(USART1)
        ▼
STM32F103C8T6
   ├── Bootloader(16KB):校验 → 备份 → 搬运 → 跳转 / 回滚
   ├── App(48KB):业务代码 + 接收新固件写入 W25Q64
   └── SPI1 ──> W25Q64(下载区 / 备份区 / 参数区)
```

升级流程:
1. App 收到上位机升级指令,把新固件分包写入 W25Q64 **下载区**,逐包 CRC 校验,OLED 显示进度;
2. 接收完成后在 **参数区** 写升级标志(含固件大小、整包 CRC32、版本号),软复位;
3. Bootloader 检测到升级标志 → 校验下载区整包 CRC32 → 把当前 App 备份到 **备份区** → 擦写内部 Flash App 区 → 清标志 → 跳转;
4. 新 App 启动自检失败 / 看门狗复位 → Bootloader 从备份区**回滚**;
5. 上电按住按键 → 强制停留 Bootloader(救砖入口)。

## 2. 内部 Flash 分区(64KB,页大小 1KB)

| 区域 | 起始地址 | 大小 | 说明 |
|---|---|---|---|
| Bootloader | 0x08000000 | 16KB (0x4000) | ST-Link 烧录,平时不变 |
| App | 0x08004000 | 48KB (0xC000) | OTA 更新的业务固件 |

- App 工程需重定位中断向量表:`SCB->VTOR = 0x08004000;`
- Keil 中通过 Options for Target → Target 页修改 IROM1 实现分区,无需手写 sct。

## 3. W25Q64 分区(8MB,扇区 4KB / 块 64KB)

| 区域 | 起始地址 | 大小 | 说明 |
|---|---|---|---|
| 参数区 | 0x000000 | 64KB | 升级标志、版本号、固件大小、CRC32(实际只用 1~2 个扇区) |
| 下载区 | 0x010000 | 64KB | 上位机推送的新固件暂存 |
| 备份区 | 0x020000 | 64KB | 升级前的旧固件,用于回滚 |
| 空闲 | 0x030000~ | ~7.8MB | 预留(日志、多版本存档等) |

## 4. 硬件接线(面包板)

| 模块引脚 | STM32 引脚 | 说明 |
|---|---|---|
| W25Q64 CS | PA4 | 软件控制 GPIO |
| W25Q64 CLK | PA5 | SPI1_SCK |
| W25Q64 DO | PA6 | SPI1_MISO |
| W25Q64 DI | PA7 | SPI1_MOSI |
| W25Q64 VCC/GND | 3.3V / GND | **必须 3.3V** |
| USB 转串口 TXD | PA10 (USART1_RX) | 交叉连接,共地 |
| USB 转串口 RXD | PA9 (USART1_TX) | |
| OLED SCL / SDA | PB6 / PB7 (I2C1) | 仅 App 工程使用 |
| 按键 | PB0 → GND | 内部上拉,按下为低(强制进 Boot) |
| LED | PC13(板载) | 状态指示 |

## 6. CubeMX 配置步骤

### 6.1 Bootloader 工程(精简,目标 < 16KB)

1. 新建工程,芯片选 **STM32F103C8Tx**;
2. **SYS**:Debug → Serial Wire;
3. **RCC**:HSE → Crystal/Ceramic Resonator;Clock Configuration 页把 SYSCLK 配到 **72MHz**(PLL ×9);
4. **USART1**:Mode → Asynchronous;115200, 8N1;暂不开中断(Boot 阶段用轮询即可);
5. **SPI1**:Mode → Full-Duplex Master;Prescaler 设为 8(约 9MHz);Data Size 8 Bits, MSB First, CPOL=Low, CPHA=1Edge;
6. **GPIO**:PA4 → GPIO_Output(W25Q64 片选,初始 High);PC13 → GPIO_Output;PB0 → GPIO_Input + **Pull-up**;
7. **Project Manager**:工程名 `Bootloader`,路径选 `OTA_project\`,Toolchain 选 **MDK-ARM**;Code Generator 页勾选 *Generate peripheral initialization as a pair of '.c/.h' files*;
8. 生成代码后,在 Keil 中:**Options for Target → Target**,IROM1 改为 `Start: 0x8000000, Size: 0x4000`。

### 6.2 App 工程

1~3 同上;
4. **USART1**:同上,另外在 NVIC Settings 勾选 **USART1 global interrupt**(后续 DMA+空闲中断收固件包,阶段三再加 DMA);
5. **SPI1**、**GPIO**:同 Bootloader;
6. **I2C1**:Mode → I2C(PB6/PB7),标准速率即可(OLED);
7. **Project Manager**:工程名 `App`,其余同上;
8. Keil 中 IROM1 改为 `Start: 0x8004000, Size: 0xC000`;
9. `main.c` 的 `USER CODE BEGIN 1` 区第一行加:`SCB->VTOR = 0x08004000;`
10. 生成 bin 文件(供 OTA 发送):**Options for Target → User → After Build** 勾选 Run #1,命令:
    `fromelf --bin -o "$L@L.bin" "#L"`

## 7. 阶段计划

| 阶段 | 内容 | 验收标准 |
|---|---|---|
| ① 双工程 + 跳转 | 建好两个工程,Boot 检查按键后跳转 App | 串口先打印 "Bootloader v1.0" 再打印 "App v1.0",App 中断正常(LED 定时闪烁);按住 PB0 上电则停留 Boot |
| ② W25Q64 驱动 | SPI 读 JEDEC ID、扇区擦除、跨页写、读回校验 | 串口打印 ID = 0xEF4017,写入→读回数据一致 |
| ③ 串口协议 + 上位机 | 帧协议(帧头/序号/长度/数据/CRC32)、Python 上位机分包发送,App 写入下载区 | 上位机发送 bin,板端逐包校验通过,OLED 显示进度 |
| ④ Bootloader 搬运 + 回滚 | Boot 校验、备份、搬运、跳转;损坏固件触发回滚 | 一键升级 v1.0→v2.0 成功;推送坏固件后自动回滚 v1.0 |

## 8. 关键技术点备忘

- **跳转前清理现场**:关闭所有已开启的中断与外设(`HAL_DeInit`)、`__disable_irq()`,设置 MSP 为 App 向量表第一个字,再跳转复位向量;App 入口处 `SCB->VTOR` 重定位后再 `__enable_irq()`。
- **跳转合法性检查**:跳转前校验 App 栈顶值是否落在 SRAM 范围(0x20000000~0x20005000),避免跳进空 Flash。
- **内部 Flash 写入**:C8T6 页大小 1KB,`HAL_FLASH_Unlock` → 按页擦除 → 半字/字编程。
- **CRC32**:与 Python `zlib.crc32` 保持一致(多项式 0x04C11DB7,反射 + 终值取反),便于上位机/板端互验。
- **升级标志**:用魔数(如 0x5A5AA5A5)+ 字段结构体存参数区,Boot 读到魔数才进升级分支,处理完立即擦除,防止反复升级死循环。
