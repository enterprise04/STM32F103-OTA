# 阶段① 操作指南:双工程 + Bootloader→App 跳转

> 前置:已按 [01_总体设计.md](01_总体设计.md) 第 6 节用 CubeMX 生成 `Bootloader` 和 `App` 两个 MDK-ARM 工程。
> 本阶段只需接:USB 转串口(PA9/PA10,共地)、按键 PB0→GND。W25Q64/OLED 可以先不接。

## 1. 两个工程共同的 Keil 设置

1. **Options for Target → Target**:勾选 **Use MicroLIB**(printf 重定向需要);
2. **IROM1 分区**(同一页):
   - Bootloader 工程:`Start 0x8000000, Size 0x4000`
   - App 工程:`Start 0x8004000, Size 0xC000`
3. **添加公共文件**:Project 窗口右键 → Add Group 新建组 `Common`,Add Existing Files 加入:
   - 两个工程都加:`..\Common\retarget.c`
   - 仅 Bootloader 加:`..\Common\boot.c`
4. **头文件路径**:Options for Target → C/C++ → Include Paths 添加 `..\Common`;
5. **App 工程生成 bin**(Bootloader 不需要):Options for Target → User → After Build 勾选 Run #1,命令:
   ```
   fromelf --bin -o "$L@L.bin" "#L"
   ```
6. **下载设置**:Options for Target → Debug → ST-Link → Settings → Flash Download,勾选 *Reset and Run*。
   注意:**用 ST-Link 给 App 工程下载时不要选 "Erase Full Chip"**,用默认 *Erase Sectors* 即可,否则会把 Bootloader 擦掉。

## 2. Bootloader 工程:main.c 代码片段

以下代码全部写在 CubeMX 的 USER CODE 区内,重新生成代码不会丢失。

```c
/* USER CODE BEGIN Includes */
#include <stdio.h>
#include "boot.h"
/* USER CODE END Includes */
```

`main()` 函数中,外设初始化完成之后:

```c
  /* USER CODE BEGIN 2 */
  printf("\r\n[Boot] Bootloader v1.0 @0x%08lX\r\n", (unsigned long)BOOT_ADDR);

  /* 上电按住 PB0:强制停留 Bootloader(救砖入口) */
  if (HAL_GPIO_ReadPin(GPIOB, GPIO_PIN_0) == GPIO_PIN_RESET) {
    printf("[Boot] Key pressed, stay in bootloader.\r\n");
  } else if (Boot_AppValid(APP_ADDR)) {
    printf("[Boot] Jump to App @0x%08lX ...\r\n", (unsigned long)APP_ADDR);
    HAL_Delay(50);              /* 等串口发完 */
    Boot_JumpToApp(APP_ADDR);   /* 正常情况下不返回 */
    printf("[Boot] Jump failed!\r\n");
  } else {
    printf("[Boot] No valid App found.\r\n");
  }
  /* USER CODE END 2 */
```

主循环里加个快闪表示"停留在 Boot":

```c
    /* USER CODE BEGIN 3 */
    HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_13);
    HAL_Delay(100);
  }
  /* USER CODE END 3 */
```

## 3. App 工程:main.c 代码片段

```c
/* USER CODE BEGIN Includes */
#include <stdio.h>
/* USER CODE END Includes */
```

`main()` 最开头(`HAL_Init()` 之前)——**这是 App 能正常响应中断的关键**:

```c
  /* USER CODE BEGIN 1 */
  SCB->VTOR = 0x08004000UL;   /* 中断向量表重定位到 App 区 */
  __enable_irq();             /* Bootloader 跳转前关了全局中断 */
  /* USER CODE END 1 */
```

外设初始化之后:

```c
  /* USER CODE BEGIN 2 */
  printf("\r\n[App] Firmware v1.0 running.\r\n");
  /* USER CODE END 2 */
```

主循环慢闪(和 Boot 的快闪区分):

```c
    /* USER CODE BEGIN 3 */
    HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_13);
    HAL_Delay(500);             /* HAL_Delay 依赖 SysTick 中断,能闪说明中断正常 */
  }
  /* USER CODE END 3 */
```

## 4. 烧录与验收

1. 先编译并用 ST-Link 下载 **Bootloader** 工程;此时串口应打印 `No valid App found.`,LED 快闪;
2. 再编译并下载 **App** 工程(确认下载设置是 Erase Sectors);
3. 按复位键,串口助手(115200)应依次打印:
   ```
   [Boot] Bootloader v1.0 @0x08000000
   [Boot] Jump to App @0x08004000 ...
   [App] Firmware v1.0 running.
   ```
   LED 变为 500ms 慢闪 → **中断向量重定位成功**;
4. 按住 PB0 再按复位:停留 Bootloader,LED 快闪 → 救砖入口正常。

全部通过即阶段①完成,进入阶段②(W25Q64 驱动)。

## 5. 常见坑

| 现象 | 原因 |
|---|---|
| 跳转后死机/HardFault | App 没改 IROM1 起始地址,或没设 `SCB->VTOR` |
| App 里 HAL_Delay 卡死 | 没有 `__enable_irq()`,SysTick 中断进不来 |
| printf 不输出 | 没勾 MicroLIB,或 retarget.c 没加进工程 |
| 下载 App 后 Boot 消失 | 下载方式选了 Erase Full Chip |
| 串口乱码 | 时钟没配到 72MHz(检查 CubeMX 时钟树)或波特率不符 |
